# HEARTBEAT

## Purpose
Keep Friday quietly useful without becoming noisy.

## Default Behavior
If there is no meaningful development, stay quiet.
Do not interrupt Alex just to report that nothing changed.

## What To Check
Rotate through these lightly during heartbeats:
- Telegram connection and operational continuity when relevant
- workspace organization and important config drift
- whether there is an obvious unfinished priority in current files
- whether a recent idea, plan, or setup needs consolidation into memory or priorities

## When To Act
Act during a heartbeat when one of these is true:
- something important is broken
- an unfinished setup can be completed safely
- a useful cleanup or organization step is obvious
- an important follow-up was implied but not completed
- a priority can be advanced without needing Alex

## When To Notify Alex
Only interrupt Alex when there is:
- a completed meaningful result
- a real blocker
- a decision that needs his input
- a meaningful risk
- an opportunity worth acting on

## Ongoing Responsibility
Prefer silent progress over repetitive updates.
If there is clear unfinished operational work, advance it.
If there is nothing meaningful to do, stay quiet.

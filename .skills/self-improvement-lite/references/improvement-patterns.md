# Improvement patterns

## Good examples
- Instagram setup failed because app tokens were confused with user tokens. Preserve the rule: use user token in Graph API Explorer for validation.
- A repeated command needed a direct script path because complex shell wrapping was blocked by exec preflight.
- A brand workflow improved after separating setup skills from operator skills.

## Weak examples
- “Need to be smarter next time.”
- “Maybe there is a better way somehow.”
- “Remember to be better at everything.”

## Heuristic
If a lesson can save time, prevent a repeated mistake, or improve a recurring workflow, keep it.
If it is just commentary, skip it.
